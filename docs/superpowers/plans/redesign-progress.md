- **2026-05-28 — Chunk 1 deviations (learned during impl, affect later chunks):**
  - **Material Symbols loads via a stylesheet `<link>`, NOT `next/font/google`.** That face isn't in next/font's registry for Next 15.5.9 (verified). `layout.tsx` injects the Google Fonts `<link>` (with `display=block` + a scoped `eslint-disable` for `no-page-custom-font`/`google-font-display` — both legit for an icon font), exposing the family via `--font-icons` in `globals.css`. `Icon.tsx` consumes `var(--font-icons)`. **Chunk 11 cleanup:** when removing `@mui/icons-material`, the Material Symbols `<link>` stays.
  - **`Icon` `sx` is a real `SxProps<Theme>`** (via a module-scope `styled('span')`, no per-render `Box`). Dynamic per-instance values (`fontFamily`/`fontSize`/`color`/`fontVariationSettings`) are inline `style`; `sx` is the caller-override hatch. Chose real SxProps over renaming to `style` because dev is fully agentic and agents pattern-match the `sx` convention (#89 review call). Decorative-by-default (`aria-hidden`); pass `aria-label` to make it `role="img"`.
  - **MUI v7 rejects custom typography variant names as `MuiTypography.styleOverrides` slots** (`displayXl` etc. → TS2353). That belt-and-suspenders override block was dropped; the typography _variants_ themselves carry `fontVariantNumeric: 'tabular-nums'`, so numeric tabular rendering is preserved. Don't re-add a `MuiTypography` slot override for custom variants.
  - **Signed-out landing page (`src/app/page.tsx`) still uses hardcoded LIGHT colors** — only the `lightTheme`→`darkTheme` import was swapped to compile. It will look light/mismatched until **Chunk 10** redesigns it. Expected interim wart; don't "fix" piecemeal.
  - **#88 filed** (middleware `pathname.includes('.')` defense-in-depth bypass) — deferred from Chunk 1; not an ownership hole (per-route `getServerSession` backstops). Revisit with route-level approval enforcement.
